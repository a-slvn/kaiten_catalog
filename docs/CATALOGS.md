# Каталоги (Catalogs)

> Каталоги — это справочники с произвольной структурой полей. Они позволяют хранить структурированные данные (например, список продуктов, тарифов, офисов) и прикреплять записи из этих справочников к карточкам сделок, контактов и компаний через пользовательские поля.

## Обзор фичи

Каталог состоит из двух уровней:

- **Схема каталога** — набор полей (как колонки в таблице), настраивается один раз при создании.
- **Записи каталога** — строки данных, которые заполняются по этой схеме.

Каталог подключается к карточке через **пользовательское поле типа «Каталог»**. После подключения пользователь может выбрать одну или несколько записей из каталога прямо в карточке.

**Где находится в интерфейсе:**
- Страница каталогов: `Настройки → Каталоги`
- Пользовательские поля: `Настройки → Пользовательские поля`
- Карточка: раздел «Пользовательские поля» в любой карточке (сделка, контакт, компания)

## Страница каталогов

Путь: `Настройки → Каталоги` (`activeItem = 'catalogs'`)

Компонент: `src/components/CatalogsPage.tsx`

### Что отображается

Таблица со всеми созданными каталогами. Колонки:

| Колонка | Описание |
|---|---|
| Название | Имя каталога. Кликабельно — открывает страницу деталей |
| Количество полей | Число полей в схеме каталога |
| Количество записей | Число записей в каталоге |
| Дата создания | Дата и время создания |
| Действия | Кнопки «Редактировать» и «Удалить» |

Если каталогов нет — отображается пустое состояние с кнопкой «Создать каталог».

### Действия на странице

**Создать каталог** (кнопка в правом верхнем углу)
- Открывает диалог создания каталога (`CreateCatalogDialog` в режиме создания)

**Редактировать каталог** (иконка карандаша в строке)
- Открывает диалог редактирования с заполненными данными

**Удалить каталог** (иконка корзины в строке)
- Показывает модалку подтверждения
- В тексте указывается имя каталога и число записей, которые будут удалены
- **Важно:** удаление каскадное — удаляются каталог и все его записи без возможности восстановления

## Создание и редактирование каталога

Компонент: `src/components/CreateCatalogDialog.tsx`

### Форма каталога

**Название каталога** (input, обязательное)
- Должно быть заполнено — иначе кнопка «Сохранить» неактивна

**Флаги поведения** (checkboxes)

| Флаг | Описание | По умолчанию |
|---|---|---|
| Множественный выбор | Разрешить выбирать несколько записей из этого каталога в карточке | выкл |
| Разрешить редактирование записей | Разрешить создание и редактирование записей прямо из карточки | выкл |

**Список полей**

Обязателен хотя бы один. Порядок полей можно менять перетаскиванием. Для каждого поля задаются:
- Название поля (обязательное)
- Тип поля (обязательное)
- Дополнительные настройки в зависимости от типа

### Добавление полей

Есть два способа добавить поле:

**1. Новое поле**

Открывается меню выбора типа. Доступные типы:

| Тип | Лейбл в UI | Описание |
|---|---|---|
| `text` | Строка | Текстовое поле |
| `url` | Ссылка | URL-адрес, отображается как кликабельная ссылка |
| `email` | Email | Email-адрес, отображается как mailto-ссылка |
| `phone` | Телефон | Номер телефона |
| `numeric` | Число | Числовое значение |
| `catalog_ref` | Каталог | Ссылка на запись из другого каталога |

Для типа `catalog_ref` дополнительно:
- Выпадающий список для выбора целевого каталога (обязательное)
- Флаг «Множественный выбор» — разрешить выбирать несколько записей

**2. Из пользовательских полей**

Позволяет использовать существующее пользовательское поле как поле каталога. Из автокомплита выбирается нужное поле — тип маппится автоматически:

| Тип пользовательского поля | Тип поля каталога |
|---|---|
| `string` | `text` |
| `number` | `numeric` |
| `catalog` | `catalog_ref` |
| `reference` | `reference` |
| `select` | `select` |
| `multiselect` | `multiselect` |
| `date` | `text` |

### Блокировка изменений

После того как у поля появились данные (хотя бы в одной записи), следующие свойства **нельзя изменить**:

- Тип поля (`type`)
- Варианты значений (`options`)
- Целевой справочник (`referenceId`, `targetCatalogId`)
- Флаг множественного выбора (`multiple`)

Можно менять: название, флаг `required`. Поле можно удалить только если в нём нет данных.

### Валидация перед сохранением

- Название каталога не пустое
- Добавлено хотя бы одно поле
- У поля типа `reference` выбран `referenceId`
- У поля типа `catalog_ref` выбран `targetCatalogId`
- У поля из пользовательских полей — выбрано конкретное поле

## Страница деталей каталога

Путь: после клика на название каталога (`activeItem = 'catalog-detail'`)

Компонент: `src/components/CatalogDetailPage.tsx`

### Что отображается

Таблица со всеми записями каталога. Колонки соответствуют полям схемы.

**Хлебные крошки** в шапке: `Каталоги → [Название каталога]`

**Особенности отображения значений:**

| Тип поля | Как отображается |
|---|---|
| `text` | Текст |
| `email` | Ссылка `mailto:` |
| `url` | Кликабельная ссылка |
| `phone` | Текст |
| `numeric` | Число |
| `catalog_ref` | `displayValue` записи целевого каталога |
| `reference` | `displayValue` связанной записи |

Если значение не заполнено — показывается `-`.
Если несколько значений (multiple) — объединяются через `, `.

### Инструменты на странице

**Поиск** (строка поиска)
- Ищет по `displayValue` и по всем полям записи
- Поиск регистронезависимый

**Сортировка**
- Клик по заголовку колонки — сортировка по возрастанию
- Повторный клик — по убыванию
- Третий клик — сброс сортировки

**Видимость колонок** (иконка настроек)
- Можно включить/выключить отображение любого поля
- Состояние хранится в памяти компонента (не сохраняется между сессиями)

### Действия с записями

**Добавить запись** (кнопка «Добавить запись»)
- Открывает `CreateCatalogEntryDialog` в режиме создания

**Просмотр записи** (клик по строке или иконка «открыть»)
- Открывает `CatalogEntryDetailDialog` с полными данными записи
- В диалоге есть вкладки: поля, связанные записи, использование в карточках

**Редактировать запись** (иконка карандаша)
- Открывает `CreateCatalogEntryDialog` с заполненными данными

**Удалить запись** (иконка корзины)
- Показывает модалку подтверждения
- Удаляет только запись, ссылки на неё из других каталогов или карточек не очищаются

## Пользовательские поля типа «Каталог»

Компонент: `src/components/AddCustomFieldModal.tsx`

### Настройка поля

При выборе типа `Каталог` в форме создания пользовательского поля появляются дополнительные настройки:

**Выбор каталога** (select, обязательное)
- Выпадающий список всех доступных каталогов
- Без выбора каталога кнопка «Сохранить» неактивна

**Множественный выбор** (checkbox)
- Разрешает выбирать несколько записей из каталога в карточке
- Флаг `isCatalogMultiple` сохраняется в определение поля

**Недоступные опции для типа «Каталог»:**
- «Пользователи могут добавлять значения» — скрыто
- «Значения могут иметь цвет» — скрыто
- Палитра цветов — скрыта

### Смена каталога при редактировании

Если у поля уже есть тип «Каталог» и пользователь меняет выбранный каталог:

1. Показывается диалог предупреждения
2. Текст: изменение каталога приведёт к очистке значений этого поля во всех карточках
3. При подтверждении:
   - Все значения поля удаляются из всех записей (`crm_deal_values_*` в localStorage)
   - Поле обновляется с новым `catalogId`
4. При отмене — каталог не меняется

### Данные, сохраняемые в определении поля

```
{
  type: 'catalog',
  catalogId:       ID выбранного каталога,
  catalogName:     Название каталога (для отображения),
  isCatalogMultiple: true/false
}
```

## Каталог в карточке

Компоненты: `src/components/DealModal.tsx`, `CatalogFieldSelect.tsx`, `CatalogFieldDisplay.tsx`

### Отображение поля в пустом состоянии

Если ни одна запись не выбрана — показывается кнопка:

```
[ Выбрать из [Название каталога] ]
```

### Выбор записей из каталога

При клике на кнопку или поле открывается автокомплит (`CatalogFieldSelect`):
- Показывает список записей каталога
- Поддерживает поиск по тексту
- Если у каталога `isMultiple = true` или `isCatalogMultiple = true` — можно выбрать несколько
- Если у каталога `isEditable = true` — в списке доступна кнопка **«+ Создать»**

### Создание новой записи из карточки

Доступно если у каталога включён флаг **«Разрешить редактирование записей»** (`isEditable = true`).

1. В выпадающем списке есть кнопка «+ Создать»
2. Открывается `CreateCatalogEntryDialog`
3. После сохранения запись автоматически добавляется в каталог и выбирается в поле

### Отображение выбранных записей

Выбранные записи показываются как карточки с `displayValue` и дополнительными полями (`CatalogFieldDisplay`):

- Показывается не более **3 записей** — остальные скрыты под кнопкой «Ещё N»
- При множественном выборе отображается чип «X выбрано»
- Каждая карточка имеет кнопку удаления (✕)

### Взаимодействие с карточкой записи

Каждая выбранная запись отображается в виде карточки с тремя зонами взаимодействия:

| Действие | Результат |
|---|---|
| Клик по карточке (в любом месте кроме кнопок) | Разворачивает / сворачивает поля записи (inline collapse) |
| Кнопка **глаза** (`VisibilityIcon`) | Открывает `CatalogEntryDetailDialog` с полными данными записи |
| Кнопка **удаления** (✕) | Снимает выбор записи (не удаляет из каталога) |

> Кнопка глаза видна всегда, вне зависимости от флага `catalog.isEditable`.

**Развёрнутое состояние** показывает все текстовые поля записи (email, phone, text, numeric, url). Поля типа `reference` и `catalog_ref` в inline-view не отображаются — они доступны в полной модалке.

### Просмотр детали записи (`CatalogEntryDetailDialog`)

Открывается по клику на иконку глаза в карточке записи. Содержит:

- Вкладка **«Обзор»** — ключевые поля записи
- Вкладка **«Карточки»** — список сделок/карточек, в которых используется эта запись

Поля типа `email` и `url` отображаются как кликабельные ссылки.
Поля типа `catalog_ref` — ссылки на записи другого каталога (можно перейти).

**Навигация по хлебным крошкам:**
```
[Карточка] → [Запись каталога] → [Запись вложенного каталога]
```

### Редактирование записи из карточки

Доступно если у каталога включён флаг `isEditable`.

В детальном просмотре (`CatalogEntryDetailDialog`) есть кнопка «Редактировать» — открывает `CreateCatalogEntryDialog`.

### Удаление записи из каталога (`CatalogEntryDetailDialog`)

Кнопка «Удалить» в нижнем левом углу модалки удаляет запись **из каталога** (не только снимает выбор).

Перед удалением показывается диалог подтверждения:
> «Запись "[Название]" будет удалена из каталога без возможности восстановления. Это действие нельзя отменить.»

### Снятие выбора записи (из карточки)

- Кнопка ✕ на карточке записи снимает выбор (не удаляет запись из каталога)
- Запись остаётся в каталоге, просто больше не привязана к этой карточке

### Хранение значений

Выбранные записи хранятся в localStorage:
- Ключ: `crm_deal_values_[dealId]`
- Значение: объект `{ [fieldId]: entryId | entryId[] }`

## Типы данных

### Каталог (Catalog)

```typescript
interface Catalog {
  id: string;          // Формат: 'catalog-[timestamp]-[random]'
  name: string;        // Название каталога
  fields: CatalogFieldDef[];  // Схема полей
  isMultiple?: boolean;       // Разрешить множественный выбор в карточках
  isEditable?: boolean;       // Разрешить редактирование из карточек
  createdAt: string;
  updatedAt: string;
}
```

### Поле каталога (CatalogFieldDef)

```typescript
interface CatalogFieldDef {
  id: string;
  name: string;
  type: 'text' | 'url' | 'email' | 'phone' | 'numeric'
      | 'select' | 'multiselect' | 'reference' | 'catalog_ref';
  required: boolean;
  options?: string[];          // Для select/multiselect
  referenceId?: string;        // Для type='reference'
  referenceName?: string;
  multiple?: boolean;          // Множественный выбор для reference/catalog_ref
  targetCatalogId?: string;    // Для type='catalog_ref'
  targetCatalogName?: string;
  customFieldId?: string;      // Если поле создано из пользовательского поля
}
```

### Запись каталога (CatalogEntry)

```typescript
interface CatalogEntry {
  id: string;          // Формат: 'catentry-[timestamp]-[random]'
  catalogId: string;   // ID родительского каталога
  displayValue: string;// Основное отображаемое значение (из первого поля)
  fields: CatalogFieldValue[];
  createdAt: string;
  updatedAt: string;
}

interface CatalogFieldValue {
  fieldId: string;
  value: string | string[] | number | null;
}
```

### Пользовательское поле типа «Каталог» (CustomFieldDefinition)

```typescript
interface CustomFieldDefinition {
  id: string;
  name: string;
  type: 'catalog';
  catalogId: string;         // ID привязанного каталога
  catalogName: string;       // Название каталога (денормализовано)
  isCatalogMultiple?: boolean; // Множественный выбор
  showOnCardFacade: boolean;
  // ... другие общие поля
}
```

## Хранение данных

Всё хранится в `localStorage` браузера.

| Ключ | Что хранится |
|---|---|
| `crm_catalogs` | JSON-массив всех каталогов (схемы) |
| `crm_catalog_entries` | JSON-массив всех записей каталогов |
| `crm_deal_values_[dealId]` | Значения пользовательских полей (включая ID выбранных записей каталогов) |
| `crm_custom_fields` | Определения пользовательских полей (включая поля типа catalog) |

**Контекст данных:** `src/context/CatalogsContext.tsx`

| Метод | Описание |
|---|---|
| `addCatalog(data)` | Создаёт новый каталог, возвращает ID |
| `updateCatalog(id, updates)` | Обновляет схему или флаги каталога |
| `deleteCatalog(id)` | Удаляет каталог и каскадно все его записи |
| `getCatalog(id)` | Возвращает каталог по ID |
| `addEntry(data)` | Создаёт запись, возвращает ID |
| `updateEntry(id, updates)` | Обновляет значения полей записи |
| `deleteEntry(id)` | Удаляет запись |
| `getEntriesByCatalog(catalogId)` | Возвращает все записи каталога |
